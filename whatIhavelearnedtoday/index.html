What is the dash or double dash represent in command line?
So today I found information on why dash and double dash are used and the purpose for their parsing.
1. If argument starts with a single -, only the single character alias is considered. a. If the named argument alias (a) needs a VALUE, it can be provided in these ways: -a=VALUE -a VALUE b. If the argument is a flag, additional characters after are treated as if they start with a -, repeatedly until the end of the argument is reached. Fails if non-flags hit. 2. If the argument starts with a double --, only the full name is considered. a. If the named argument (arg) needs a VALUE, it can be provided in these ways: --arg=VALUE --arg VALUE 3. If the argument does not start with any -, it is considered the next positional argument. 4. If the argument is only a double --, all further arguments are only considered as positional.
So, in effect the single dash represents the short hand and when used depending on the developer/team/organization the rest of the command to the following space is ignored. but when double dash is used the entire word to the next word boundary is used and parsed.
Additionally, within this specification you can only have 26 single dashed commands but (nearly)unlimited double dashed commands.
